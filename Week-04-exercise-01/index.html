My Dev Tools Vandalism

I used browser developer tools to temporarily "vandalize" a website. Below is a screenshot showing my changes. These edits only appeared on my computer and did not affect the actual website.

Screenshot of the Webpage I Vandalized

What I "Vandalized"

For this exercise, I used the browser's developer tools to change a website homepage. I edited the main words on the page and changed them to say “Welcome to Slimeto University.”

I also changed the CSS styles on the page. I changed the main words to red and changed the background behind the words to green. These changes made the page stand out and look completely different from the original website.

My edits were interesting because they made the website look like it had been turned into my own fake university page. The red words with the green background made it look bold, funny, and easy to notice.

Some changes worked right away because I selected the correct HTML element and edited the text directly. Some changes were a little harder because websites can have many CSS rules, and sometimes another rule controls the color, size, or background.

I chose these changes because I wanted the page to look creative and personal. Changing the text to “Welcome to Slimeto University” made the website feel like my own version, and changing the colors made the edit more obvious in the screenshot.

This exercise helped me understand how HTML and CSS work together. The HTML controls the words and structure of the page, while the CSS controls how those words look, including the color and background.

I now have a better sense of how developer tools can help me while making my own webpages. I can use dev tools to test changes, experiment with styles, and understand what part of the code controls each section of a page.
